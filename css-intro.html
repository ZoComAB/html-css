Selektorer

Anger vad som ska stylas

Det är i css som "styrkan" anges

header{} /* väljer alla taggar */
.header{} /* väljer alla taggar med klassen "header" */
#header{} /* väljer elementet med id "header" */

Selektorer kan vara ännu mer specifika

"Jag vill välja alla a-taggar inuti 'header'"

.header a {
  color: red;
}

Observera mellanslaget mellan header och a

Går att välja hur djupt som helst

"Jag vill välja alla p-taggar i alla a-taggar inuti 'header'"

.header a p {
  color: red;
}

Observera mellanslaget mellan header och a

Klasser, id och taggar går att kombinera

"Välj alla a-taggar inuti klassen header inuti id main-container

#main-container .header a {
  color: red;
}

Egenskaper

Måttenheter

.header {
  width: 200px;
  width: 20rem; /* relativ  till fontstorleken */
  width: 20%; /* relativ till förälderns bredd */
  width: 20vw; /* relativ till fönstret: 1vw = 1% av fönstrets bredd*/
}

Många egenskaper utgår från vad föräldern har för egenskaper

<div> <!-- Förälder -->
  <p> Child   </p>
  <p> Child 2 </p>
<div>

Om inget är skrivet så ärver elementet av föräldern: Cascading

Min och max sätter höjd och bredd om man inte vet exakt hur stort det ska vara

.header{
  max-width: 300px;
  width: 200px;     
}

max-width sätter inte själva bredden

Går även på height

Boxmodellen

bounding, margin & padding

Varje element utgår från denna modell

De flesta element är av typenblock

block har width och tar upp hela bredden

Resten är antingen inline

inline har ingen bredd

margin är avståndet utåt till andra element

padding är avståndet inåt till andra element

Avstånd mellan element

.header{
  margin: 30px; /* 30px på alla sidor */
  margin: 30px 10px 10px 10px;
}

Stter vi fler värden så kan vi bestämma alla sidor

Top -> right -> bottom -> left

Många egenskaper har Shorthands

.header{
  margin: 30px; /* shorthand för alla sidor */
  border: 1px solid red;
}

Med border: sätter vi border-widthborder-style och border-color

position

Med position bestämmer vi delvis var på sidan elementet ska vara positionerat

Default är static och säger inte så mycket

.header{
  position: relative;
  top: 50px;
  left: 50px;
}

Med Relative menas relativ till sig själv.

Absolut positionering

.header{
  position: absolute;
  top: 50;
}

Oftast i relation till <body>

Fixed

.header{
  position: fixed;
  top: 50px;
  right: 50px;
}

Följer vår "viewport", följer med när vi scrollar

Oftast enklaste sättet att positionera något i mitten

.header{
  width: 300px; /* Bredd måste finnas */
  margin: auto; /* Marginalen expanderar automatiskt */
}

Generell regel:

Undvik fasta mått, alla har inte samma storlek på skärmen

Dock så är det enklaste att göra fasta mått

float

Ett standardsätt att positionera element bredvid varandra

Block är inställt på att ta upp hela bredden

.box{
  width: 300px; /* Måste ha en bredd */
  float: right;
}
.box2{
  width: 300px; /* Måste ha en bredd */
  float: left;
}

Floats kan ta över, fortsätta flyta grejer fast vi inte vill

Då får vi använda egenskapen clear

.footer {
  clear: both;      
}

När vi har använt float på ett element så kan vi säga på vilken sida det INTE får fortsätta flyta på

.box{
  clear: left;
  clear: right;
  clear: both;
}
